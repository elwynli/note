# 软件需求

触发器、参与者、动作、对象、条件

## 什么是好需求

1. 正确
2. 一致
3. 可行
4. 必须
5. 有优先级
6. 完整
7. 明确
8. 与实现无关
9. 可验证
10. 原子性
11. 唯一性
12. 可修改
13. 可追溯
14. 积极描述
