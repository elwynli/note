# 度量指标

软件度量指标的基本事实是没有完美指标，但聊胜于无。

## 度量指标举例

### 1.可执行文件大小

使用汇编的程序要小得多。

### 2.机器指令度量指标

这个指标和编程语言的实现相关。

### 3.代码行数

又叫 LOC / KLOC，最常见指标。

优点：

- 统计简单
- 编程语言无关
- 不受依赖库影响
- 与CPU无关

缺点：

- 反映做了什么，但不能很好反映实现了什么

### 4. 语句行数

### 5. 功能点分析

FPA (Function Point Analysis)。

考虑程序需要输入数量和输出数量，以及执行的基本计算。

优点：独立于语言。

缺点：不能自动计算。

不过，一般用于事后统计。

### 6. McCabe 圈复杂度

它视无数行 printf 为一行，计算流程图中的节点数、边数和连接的组件数量。

优点：能自动统计

缺点：和 LOC 不相关

## 度量指标的问题

LOC 好不到哪去，但也不差。

### 我们每天可以产出多少代码

也许有人说一天，可以百八十行，但从项目经验上看，拉到整个项目时，一个人平均每日产出 10 行代码。
